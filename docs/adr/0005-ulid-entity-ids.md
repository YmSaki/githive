# ADR-0005: エンティティ ID とイベント ID に ULID を使う

- 状態：採用
- 日付：2026-07-04

## 文脈

分散環境（オフラインの複数クライアント）で衝突しない ID が必要である。
ID は ref 名、ファイル名、イベント順序の 3 役を兼ねる。

## 決定

ULID を小文字 Crockford Base32 の 26 文字で使う。
イベントの全順序は ULID の辞書順とする。
人間向けの連番は v1 では持たず、forge モードのオプション機能（counters ref）として設計余地だけ残す。

## 理由

- ULID は時刻先頭 48 bit により辞書順 ≒ 時刻順で、ファイル名にしただけで時系列に並ぶ。UUIDv4 にはこの性質がない。
- UUIDv7 も同等の性質を持つが、Base32 の 26 文字（ULID）はハイフンなしで ref 名・ファイル名に安全で、短縮前方一致とも相性がよい。
- 中央連番は分散生成できず、hosted モードに発行者がいない。

## 帰結

- 利用者は `#123` の代わりに短縮 ULID（先頭 8 文字）とタイトルで参照する。CLI は前方一致解決を提供する。
- 小文字固定とする。git の ref 名は大文字小文字を区別するが、Windows のファイルシステムは区別しないことがあるため、混在を許すと事故になる。
- ULID のランダム部により、同一ミリ秒の複数イベントも一意に順序付く。
